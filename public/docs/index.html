Introduction

Base URL: http://travel-api.test

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

Auth

POST api/v1/login

Example request:
curl --request POST \
    "http://travel-api.test/api/v1/login" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"email\": \"lmetz@example.com\",
    \"password\": \"VjM-:y-=r(wGO_\"
}"

const url = new URL(
    "http://travel-api.test/api/v1/login"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "email": "lmetz@example.com",
    "password": "VjM-:y-=r(wGO_"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

    "token": "1Aytfsfy45kJDSDSD"
}

Headers

Content-Type      

Example: application/json

Accept      

Example: application/json

URL Parameters

travel_slug   string   

The slug of the travel. Example: corporis-quia-minus-ullam

Body Parameters

priceFrom   integer  optional  

Must be at least 0. Example: 64

priceTo   integer  optional  

Must be at least 0. Example: 46

dateFrom   string  optional  

Must be a valid date. Must be a date after or equal to today. Example: 2082-12-16

dateTo   string  optional  

Must be a valid date. Must be a date after or equal to today. Example: 2084-05-15

sortByPrice   string  optional  

Must be one of ASC or DESC. Example: ASC
